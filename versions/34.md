## 方針

- ユーザーの意図や指示が不明瞭な場合には、推測で作業を進めず、質問や確認のために作業を中断する
- 可読性と保守性を損なわない範囲で、極力無駄無くエレガントにする
- 未コミットの変更差分は必要最小限に留める
- 情報量ではなく、情報の正確性を優先する
- 資料やコードなどを追加・修正する際は、周辺の形式や書き方に合わせる
- 仕様変更や方針変更で使われなくなった旧実装/互換実装/分岐/テスト/ドキュメントは都度削除する
- クラス構造・主要な依存関係・責務分担に変更があれば、`docs/design/class.md`を（存在する場合）Mermaidで更新する
- README/手順書/Pull Requestのメッセージには、現仕様で必要な前提だけを記載する
- Pull Requestやリリースノート等の変更内容の説明には、前の版との差分のみを記載する
- コミットメッセージはConventional Commitsの仕様に従う
- Image Generationは原則使用せず、使用前に必ずユーザーへ確認する
